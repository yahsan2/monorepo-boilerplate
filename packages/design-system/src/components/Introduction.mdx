# General Design System

- このデザインシステムは、サービス全体のUIを、一貫性を保ちつつ、高速に実装するためにつくられたものである。

## 特徴

- このデザインシステムでは、汎用的な機能しか持たない。
- トークンを差し替えれば、すぐにでも他のサービスで使えるような構成を目指す。
- ドメイン情報は含めず、ドメイン情報を含むコンポーネントはアプリケーション側で実装する。

## 構成

Atomic Design をベースに、デザイントークン、レイアウトプリミティブを参考にして、以下のような構成となっている。

### Tokens

- デザイントークンは、WEB サービスを構成する"値"の定義とする

### Frames

- レイアウトを構成するためのコンポーネント。
- レイアウトを組むためのフレームとしての扱いであり、必ず１つ以上の **`<Slot/>`** が存在する。

### Atoms

- それ以上に分解できない、単一の HTML 要素として、構成される。（:before, :after などの疑似要素は含まない）
- ただし `<input/>` など、擬似要素が使えない場合に限り、単一のラッパー要素を許容する。（例: InputText）

### Molecules

- 複数の HTML要素で構成される。Atomsを含む場合もあれば、別の Moleculesを含む場合もある。
